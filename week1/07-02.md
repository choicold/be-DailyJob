# be-DailyJob

> 이 회고는 작업량을 보고하기 위한 문서가 아니라, 오늘 내가 어떤 판단을 했고 어떤 개발 행동을 반복하거나 바꿔야 하는지 확인하기 위한 기록입니다.
> 각 항목은 길게 쓰기보다, 사실/판단/근거/다음 행동이 드러나게 구체적으로 작성하세요.

###### 1. 오늘 완료한 것
- 오늘 실제로 끝낸 기능/수정/학습: 
  - 데이터 모델링
  - 웹서버 2단계 다양한 컨텐츠 타입 지원
  - 웹서버 3단계 GET으로 회원가입 절반
- (오늘의 목표 중에서) 아직 끝내지 못한 부분: 
  - 웹서버 3단계 GET으로 회원가입 나머지 부분

###### 2. 가장 막혔던 문제와 해결 과정
- 문제 상황 또는 증상: 
  - `/register/index.html`에서 각 `form`에 값을 담고 회원가입을 눌렀을 때 
  `/create?userId=아이디&password=비밀번호&name=닉네임&email=이메일`형식으로 반환을 받아야 하는데 어떻게 서버에 전달할지에 대해 막혔음.
- 처음 추측한 원인: 
  - 처음에는 `RequestHandler.java`수준에서 해결해야겠다고 판단하여 접근함.
- 원인을 확인하기 위해 시도한 방법: 
  - 하지만 회원가입은 기존 `RequestHandler.java`에서 사용한 정적 파일을 불러오는 것과는 다른 형태였기에 
  이를 `/register/index.html`내의 `form`과 `input`태그의 속성을 활용하는 방안으로 시도.
- 실제 원인: 
  - 응답이 서버로 HTTP Request 형태로 넘어오기 위해서는 `/register/index.html`내의 태그들의 속성을 활용했어야 했음.
- 해결 방법: 
  - `form` 태그 내 속성인 `action=/create`과 `method=get`를 `input` 태그 내 속성인 `name=userId`을 `button` 태그 내 속성인 `type`을 `submit`으로 바꿔주어 해결했다.
- 다음에 비슷한 문제가 생기면 먼저 확인할 것: 
  - 정적 파일 불러오기가 아닌 정적 파일로부터 서버에 요청을 받을 경우에는 html 폼 태그 내 속성이 어떻게 구성되어 있는지 확인해봐야할 것 같다.

###### 3. 오늘 잘한 행동(개발과 관련된 것)
- 어떤 행동이었는지: 
  - 과제 수행 시 잘 모르는 개념에는 AI에게 빠르게 물어보고, 실제 구현에 있어서는 AI에 의존하지 않고 스스로 사고하고자 하였음.
- 그 접근이 도움이 된 이유: 
  - 잘 모르는 개념을 빠르게 보고 실제 구현 때 해당 개념을 사용하려고 애쓰다 보니 이해도 잘되고 기억도 오래 남는 느낌이다.
- 다음에도 반복하고 싶은 점: 
  - 잘 모르는 개념 위주로는 AI에게 물어보고, 실제 구현을 고민하는 과정은 최대한 스스로 사고하는 방법을 다시 반복하고 싶음.

###### 4. 오늘 아쉬웠던 행동(개발과 관련된 것)
- 비효율적이었던 접근/놓치거나 부족했던 점: 
  - 과제 구현 시 현재 `RequestHandler.java`와 `WebServer.java`에 모두 구현되어 있다.
- 그렇게 된 원인: 
  - 구현에만 포커스를 두고 리팩토링하는 과정에 대해서는 계속 미뤄와서 그렇다.
- 다음에는 다르게 해볼 구체적인 행동: 
  - 반복되거나 과한 책임이 있는 메서드의 경우에는 따로 클래스 또는 인터페이스로 나누어 코드 분리를 시도해볼 것 같다.
